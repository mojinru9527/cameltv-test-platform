# CamelTv 每日工作汇总

> 目的：每天自动汇总测试平台更新内容，蒸馏重复模式，7 天小汇总，30 天大汇总。解决「版本更新不做文档记录」的严重问题。

## 触发时机

- **自动**：UTC+8 每天 00:00（Cron：`0 0 * * *`）
- **手动**：`/cameltv-daily-summary` 或 Skill tool 激活

## 三级汇总体系

```
每日摘要 (daily-YYYY-MM-DD.md)
    ↓ 7 天聚合
每周小汇总 (weekly-YYYY-WW.md)
    ↓ 4 周聚合
每月大汇总 (monthly-YYYY-MM.md)
```

## 执行流程

### 第一步：确定时间窗口

- 检查 `work-logs/summaries/` 下最新一份摘要的日期
- 确定本次需要覆盖的日期范围（从上次摘要次日到今天）
- 如果当天已有一份，跳过（幂等）

### 第二步：收集当日变更

按以下维度扫描，每个发现附 `文件:行号`：

| 维度 | 数据源 | 提取内容 |
|------|--------|---------|
| **Git 提交** | `git log --since="YYYY-MM-DD" --until="YYYY-MM-DD+1" --oneline --no-merges` | 每条 commit 的模块、类型（fix/feat/refactor）、影响范围 |
| **Agent Team 工件** | `work-logs/batch-*-*.md` 的修改时间 | 新完成的 batch、各部门产出摘要（1-2 句话） |
| **文件变更量** | `git diff --stat` 自上次摘要 | Top 5 变更文件 + 净增/删行数 |
| **重复模式** | 对比近 3 天摘要 | 同类 bug 是否反复出现、同类改动是否重复 |

### 第三步：蒸馏重复内容

逐条检查当日变更是否命中已知模式：

- 与 `cameltv-bug-guard` 铁律对照 → 命中则标注「违反铁律 Bx/Fx」
- 与近 7 天摘要对照 → 同类改动出现 ≥2 次则标记「重复模式」，建议固化
- 与 `work-logs/` 历史 batch 对照 → 同一文件反复修改 → 标记「设计不稳定」

### 第四步：写出每日摘要

**文件**: `work-logs/summaries/daily-YYYY-MM-DD.md`

**模板**:

```markdown
# 每日摘要 YYYY-MM-DD

## 变更概览
- 提交数: N | 文件变更: N | +N/-N 行
- 涉及模块: [模块列表]

## 提交明细
| Commit | 模块 | 类型 | 说明 |
|--------|------|------|------|

## 重复模式告警
| 模式 | 命中次数 | 建议 |
|------|---------|------|

## Agent Team 进展
| Batch | 状态 | 关键产出 |
|-------|------|---------|

## 需要关注
- [ ] 待跟进项
```

### 第五步：判断是否触发周汇总

如果今天是周日（`weekday() == 6`）→ 执行**第七步**。
如果今天是月最后一天 → 执行**第八步**。

### 第六步：写周汇总（每周日）

**文件**: `work-logs/summaries/weekly-YYYY-WW.md`

聚合近 7 天每日摘要：

```markdown
# 周汇总 YYYY-WW (YYYY-MM-DD ~ YYYY-MM-DD)

## 本周数据
- 提交总数: N | 文件变更: N | 活跃天数: N/7
- 新增 Batch: N | 完成 Batch: N

## 本周重点
（3-5 条最重要的变更/决策，每条 1-2 句话）

## 重复模式深度分析
（7 天维度看重复——同类 bug ≥3 次 → 建议改流程/加测试/加固 skill）

## 模块热度
| 模块 | 变更次数 | 趋势 |
|------|---------|------|

## 下周关注
- [ ] 待办
```

### 第七步：写月汇总（每月最后一天）

**文件**: `work-logs/summaries/monthly-YYYY-MM.md`

聚合近 4 周周汇总：

```markdown
# 月汇总 YYYY-MM

## 本月数据
- 提交总数: N | 活跃天数: N/30
- 完成 Batch: N | 新增/修复 Bug: N

## 本月里程碑
（3-5 条最重要的里程碑）

## 重复模式趋势
（月度趋势图——哪些模式在恶化，哪些已根除）

## 模块健康度
| 模块 | 变更频率 | 重复 bug 数 | 健康评级 |
|------|---------|------------|---------|

## 下月重点
- [ ] 战略级待办
```

## 产出目录结构

```
work-logs/summaries/
├── daily-2026-07-19.md
├── daily-2026-07-20.md
├── ...
├── weekly-2026-W29.md
├── weekly-2026-W30.md
├── ...
├── monthly-2026-07.md
└── monthly-2026-08.md
```

## 关联

- `cameltv-bug-guard` skill — 重复 bug 模式对照源
- `cameltv-agent-team` skill — Agent Team 工件扫描源
- memory `[[current-work-focus]]` — 当前重点工作方向
- Cron 注册：每次新会话启动时检查 cron 是否存活
