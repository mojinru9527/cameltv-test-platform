# 测试用例模板与字段（来源:功能测试用例规范.md）

## 用例字段定义（以 PingCode 用例模板为准）

| # | 字段 | 说明 |
| --- | --- | --- |
| 1 | 用例编号 | 唯一标识,规则以 PingCode 用例模板为准(可系统生成,也可自定如 `V6-12-3-001`) |
| 2 | 模块名称 | 所属模块 |
| 3 | 用例标题 | 对测试项的简短描述 |
| 4 | 维护人 | 用例编写人 |
| 5 | 用例类型 | 按系统分类填写(功能/界面/性能/兼容/安全…) |
| 6 | 重要程度 | P0 / P1 / P2 |
| 7 | 测试类型 | 手动测试 / 自动测试 |
| 8 | 关联项目 | 可放入故事编号 |
| 9 | 前提条件 | 执行用例所需的预置条件 |
| 10 | 操作步骤 | 完成动作的操作,**必须明确输入数据** |
| 11 | 预期结果 | 执行后程序的表现,**含显式验证手段与判断标准** |
| 12 | 备注 | 补充说明 |
| 13 | 执行结果 | 通过 / 失败 / 阻塞 |
| 14 | 实际结果 | 实际输出 |
| 15 | 缺陷 | 出现错误时建立缺陷(见 缺陷管理制度.md) |
| 16 | 执行人 | 执行测试的人员 |
| 17 | 结果附件 | 截图等 |

## 重要程度（P0/P1/P2）

- **P0(优先执行)**:关键路径、最常用功能、基本流输入、界面数据有效性校验。全过 → 功能稳定。
- **P1(次级执行)**:不常用功能、异常流输入、边界值、异常数据。全过 → 可发布。
- **P2(最后执行)**:GUI、界面显示、错误信息提示一致性、可用性、压力性能等。

## 场景法矩阵（复杂业务流程,来源:测试用例具体用法.md）

复杂流程用矩阵管理用例:行=测试用例,列=各条件 + 预期结果。用 `V`(有效/走基本流)、`I`(无效/激活备选流)、`n/a`(不适用)标记每个条件,一眼看出覆盖是否充分。

| 用例ID | 场景/条件 | 条件A | 条件B | … | 预期结果 |
| --- | --- | --- | --- | --- | --- |
| TC1 | 场景1-成功 | V | V | | 正常完成 |
| TC2 | 场景2-条件A无效 | I | V | | 走备选流A,给出提示 |

矩阵中出现空白(无 V/I)即代表该条件缺用例,需补齐。
