## 2. 正式通知配置

### Webhook 方式

在 Web 前端进入“通知中心”，新增通道并填写：

| 配置项 | 填写内容 |
| --- | --- |
| 通道类型 | Webhook |
| 提供商 | 飞书、钉钉、企业微信或 Generic |
| Webhook 地址 | 项目群机器人或接收服务的 HTTPS 地址 |
| 订阅事件 | `task_started`、`task_finished`、`test_result` |
| 可选事件 | `plan_done`、`report_generated`、`defect_created`、`gate_failed` |
| 状态 | 配置并测试成功后启用 |

建议给平台提供单独的群机器人地址，避免与生产告警机器人混用。若地址包含签名密钥，应按敏感信息传递，不要写入需求或测试报告。

钉钉群机器人可以直接使用：在提供商选择“钉钉”，平台会按钉钉 Markdown 机器人格式发送任务发起、任务结束和测试结果。当前建议机器人安全设置选择“自定义关键词”，关键词填写“测试”；平台尚未实现钉钉 `timestamp + secret` 动态签名，如必须使用“加签”模式，需要再补充签名密钥字段和签名计算。正式联调仍需提供钉钉机器人完整 Webhook 地址。

### SMTP 方式

后端运行环境填写以下配置，密码不要提交到代码仓库：

```dotenv
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=test-platform@example.com
SMTP_PASSWORD=<应用专用密码或授权码>
SMTP_FROM=test-platform@example.com
SMTP_USE_TLS=true
SMTP_VERIFY_CERT=true
```

然后在 Web“通知中心”新增邮件通道，填写收件人列表并订阅开始、结束和测试结果事件。生产环境建议使用邮件服务的应用专用密码，不使用个人邮箱登录密码。
