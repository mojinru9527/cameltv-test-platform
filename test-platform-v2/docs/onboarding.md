# 接入向导

## 第一步：登录

1. 浏览器打开 `http://localhost` (或 `http://localhost:5173` 开发模式)
2. 使用管理员账户 `admin / admin123` 登录

## 第二步：创建项目

1. 进入「项目管理」页面
2. 点击「新建项目」，填写：
   - 项目编码（唯一标识，如 `myapp`）
   - 项目名称
   - 描述
3. 创建后在顶栏切换到此项目

## 第三步：配置角色和权限

1. 进入「系统管理 → 角色管理」
2. 创建新角色（如「开发人员」），为其分配适当权限点
3. 进入「系统管理 → 用户管理」创建新用户并分配角色

## 第四步：导入测试用例

方式一：通过 Web 界面的「用例服务」页面逐个创建
方式二：通过 API 批量导入：
```bash
curl -X POST http://localhost:8000/api/v1/test-cases \
  -H "Authorization: Bearer <token>" \
  -H "X-Project-Id: 1" \
  -H "Content-Type: application/json" \
  -d '{"title":"测试用例", "domain":"功能", "module":"登录"}'
```

## 第五步：创建测试计划

1. 进入「测试计划」
2. 点击「新建计划」，填写名称和起止日期
3. 在计划详情页添加用例

## 第六步：执行测试

1. 在计划详情中，对每条用例点击「执行」
2. 选择执行结果（通过/失败/跳过/阻塞），填写备注
3. 查看执行历史和统计

## 第七步：生成报告

1. 进入「报告中心」
2. 点击「生成报告」，选择计划
3. 报告会快照当前计划的状态和执行结果
4. 查看报告详情获取完整测试数据

## 第八步：设置定时任务

1. 进入「定时任务」
2. 点击「新建」，选择目标计划和 Cron 表达式
   - 示例：`0 9 * * 1-5` 工作日每天9点执行
   - 示例：`0 */4 * * *` 每 4 小时一次
3. 可通过 Switch 开关启禁调度
4. 也可随时手动触发

## 可选：配置 ELK 链路追踪

在 `.env` 中配置：
```
ELK_BASE_URL=https://elk.your-domain.com/app/kibana
ELK_INDEX=your-index-*
```

执行用例时，在备注中包含 `traceId: <id>`，系统会自动生成 Kibana 链接。
