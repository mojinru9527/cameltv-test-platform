# 知识中心与 Agent 持续学习功能验收报告

> 验收日期：2026-07-10  
> 验收环境：本地 `http://localhost:5173` + `http://localhost:8000`  
> 验收依据：`docs/RAG知识图谱与Agent持续学习能力落地执行文档.md`、蓝湖需求链接、测试平台当前实现  
> 验收结论：功能骨架已落地，但当前版本不能按“持续学习闭环”验收通过。主要阻断为前端构建失败、知识入库/RAG/图谱开关未启用且无启用入口、Agent 端到端执行失败、权限与队列接口不一致。

1. 知识中心入口与菜单。
2. 知识概览、知识源、RAG 检索、AI 审核台、知识图谱、实体、迭代知识包。
3. Agent 工作台、Agent 类型、手动触发、执行历史、任务队列。
4. 需求/蓝湖链接、接口导入、用例、缺陷、执行失败等知识入库 Hook 的实现接线。
5. 构建、单测、后端 API、数据库落库情况。

蓝湖链接访问情况：

- 蓝湖账号登录后能打开项目页，但页面持续出现“由于打开过分享页，用户状态发生变化，为安全起见，请重新登录”提示，无法稳定读取完整原型内容。

## 四、P0 阻断问题

### P0-1 前端发布构建失败

现象：

- `npm run typecheck` 失败：`tsconfig.json(29,18): Referenced project ... tsconfig.node.json may not disable emit`
- `npm run build` 失败：
  - `frontend/src/pages/agent-workbench/index.tsx:119`：`AgentRun[]` 不能赋值给 `AgentQueueItem[]`
  - `frontend/src/pages/knowledge/components/IterationTab.tsx:26-28`：`KnowledgeIteration`、`KnowledgeSnapshot`、`CompareSnapshots` 从 `@/api/knowledge` 导入但未导出

影响：

- 当前版本不能作为可发布版本。

调整思路：

1. 修复 `frontend/src/api/agent.ts`：`fetchQueueItems()` 返回类型应为 `KnowledgePage<AgentQueueItem>` 或新增 `AgentQueuePage`，不能复用 `AgentRunPage`。
2. 修复 `frontend/src/api/knowledge.ts`：显式 `export type { KnowledgeIteration, KnowledgeSnapshot, CompareSnapshots } from '@/types'`，或让 `IterationTab` 直接从 `@/types` 导入类型。
3. 修复 `tsconfig` 引用关系，保证 `npm run typecheck` 能独立通过。
4. 将 `npm run build` 纳入 DEV 提交流程的强制门禁。

### P0-2 Agent 端到端执行失败，无法产出 AI 审核台数据

现象：

- Agent 历史记录全部失败。
- 队列项触发后会生成失败的 `agent_run`，`ai_artifact` 始终为 0。
- 代码中 `backend/app/services/knowledge/agent_orchestrator.py:187` 创建 `AiArtifact` 时使用 `source_run_id=run.id`，但模型字段为 `agent_run_id`。

影响：

- Agent 无法把需求分析、影响分析、用例生成结果写入审核台。
- “Agent -> AI 产物 -> 人审 -> 导入正式资产”的主链路断开。

调整思路：

1. 将 `source_run_id` 改为 `agent_run_id`。
2. 单测补充“LLM 成功返回后能创建 `AiArtifact`，且 `agent_run_id` 正确关联”的用例。
3. 前端详情页展示真实失败原因，不要只显示“Agent 执行异常”。
4. 队列执行失败时保留错误码、异常类型、关联 trace_id，便于 DEV/QA 排查。

### P0-3 知识库当前没有真实数据闭环

现象：

- `knowledge_source`、`knowledge_chunk`、`knowledge_vector`、`knowledge_entity`、`knowledge_relation` 全为 0。
- `knowledge_ingest_enabled=False`、`rag_enabled=False`、`knowledge_graph_enabled=False` 为默认关闭。
- 页面没有开关状态展示，也没有“启用入库/回填/提取/验收”的操作向导。

影响：

- 平台当前无法证明具备持续学习能力。
- 新提交的蓝湖链接/需求文档不会沉淀为知识，RAG 和图谱也无法使用。

调整思路：

1. 增加“知识中心配置/运维面板”，展示并管理：
   - `knowledge_ingest_enabled`
   - `rag_enabled`
   - `knowledge_graph_enabled`
   - `agent_auto_trigger_enabled`
   - `ai_artifact_allow_batch_import`
2. 提供存量回填入口：
   - 需求文档回填为知识源
   - 接口资产回填为知识源
   - 测试用例回填为知识源
   - 缺陷与执行失败回填为知识源
3. 回填完成后展示数量变化、失败明细、脱敏样例和最近入库记录。
4. 在知识库为 0 或开关关闭时，页面显示明确引导，而不是只显示空态。

## 五、P1 高优先级问题

### P1-1 Agent 权限码不一致

现象：

- 种子权限包含 `agent:list`、`agent:run`、`agent:admin`，未包含 `agent:read`。
- 队列接口使用 `require_permission("agent:read")`。
- tester 权限有 `agent:list`，访问 `/agents/queue` 和 `/agents/queue/stats` 返回 403。
- tester 菜单没有 `/agent-workbench`。

影响：

- 普通测试人员无法使用或查看 Agent 工作台的完整状态。

调整思路：

1. 统一权限码命名，建议：
   - `agent:list`：查看运行历史和队列
   - `agent:run`：手动触发 Agent
   - `agent:admin`：管理 Agent 配置和开关
2. 或补齐 `agent:read` 并在 seed 中授权。
3. 明确 tester 是否应看到 Agent 工作台；若只读可见，应给 `menu:agent-workbench` + `agent:list`。

### P1-2 Agent 触发响应语义错误

现象：

- `POST /agents/run/{agent_type}` 返回字段名是 `run_id`，但实际返回的是 `agent_queue_item.id`。
- 队列执行时一次触发可能因重试产生多条 `agent_run`，前端无法根据返回 ID 精准追踪本次执行。

影响：

- 前端轮询、详情跳转、用户感知都会混乱。

调整思路：

1. 返回结构改为：
   ```json
   {
     "queue_item_id": 3,
     "run_id": null,
     "status": "pending"
   }
   ```
2. 队列项增加 `current_run_id` 或 `run_ids` 字段。
3. 前端触发后默认跳到“任务队列”，待运行记录生成后再关联到详情。

### P1-3 RAG/图谱错误态展示不清晰且重复 toast

现象：

- RAG 关闭时搜索返回 HTTP 503，页面同时显示“暂无相关知识”和多个错误 toast。
- 图谱关闭时接口返回 HTTP 200 + `{code:503}`，与 RAG 搜索的 HTTP 503 不一致。
- 组件 catch 和 axios 拦截器重复提示。

影响：

- 用户不知道是“没有数据”还是“功能未启用”。

调整思路：

1. 后端错误语义统一：业务不可用使用 HTTP 503 + `{code:503}`。
2. 前端区分三种状态：
   - 功能未启用
   - 已启用但无数据
   - 请求失败
3. 统一 toast 策略：全局拦截器和页面组件二选一提示，避免重复。

### P1-4 AI 审核台没有可验收闭环

现象：

- `ai_artifact` 为 0。
- 不能验证“采纳/驳回/导入正式用例库”链路。

影响：

- 治理闭环不可验收。

调整思路：

1. 修复 Agent 产物写入后，补充一条内置测试 Agent fixture 或模拟 LLM 成功返回的验收接口。
2. AI 审核台增加：
   - 产物来源
   - 关联 Agent Run
   - 引用知识片段
   - 差异预览
   - 导入结果链接
3. 增加 E2E：触发用例生成 Agent -> 生成待审产物 -> 审核通过 -> 导入测试用例 -> 用例库可见。

### P1-5 蓝湖需求入口与知识中心未串起来

现象：

- 蓝湖链接目前在“需求文档”模块提交，不在知识中心。
- 提交蓝湖链接后只保存为需求文档，当前不会入库为知识源。
- 如果蓝湖原型为图片/Axure 页面，自动提取可能失败，需要补充说明兜底。

影响：

- 用户以为提交需求链接后 Agent/RAG 会学习，但实际没有形成知识沉淀。

调整思路：

1. 知识中心增加“导入知识源”入口，支持：
   - 蓝湖/Axure 链接
   - Swagger/OpenAPI
   - Markdown/Word/Excel
   - 手工文本
2. 导入后显示处理流水线：
   - 已保存需求
   - 已提取文本
   - 已切片
   - 已脱敏
   - 已向量化
   - 已提取实体
3. 对蓝湖链接增加状态标识：
   - 已提取文本
   - 图片型原型，需要补充说明
   - 登录态失效
   - 页面权限不足

### P1-6 文档状态与实现状态不一致

现象：

- 落地文档写 M4 为规划中，但代码已经包含 Agent 触发、队列、变更检测、迭代知识包等 M4/M5/M6 能力。
- 模型文件注释仍写“knowledge_entity / knowledge_relation 留给 M3”，但实际已实现图谱 API 和前端 Tab。

影响：

- 后续验收、排期和责任边界容易混乱。

调整思路：

1. 更新落地文档阶段状态：
   - M0/M1/M2/M3：已实现但需启用验收
   - M4：部分实现，端到端不通过
   - M5/M6：试验性实现，需补验收标准
2. 每个阶段增加“完成定义”和“验收数据”。

## 九、总体结论

本次新增功能不是纯模拟页面，后端模型、API、服务、队列、入库 Hook、单元测试都已经存在；但当前版本距离“真实可用的持续学习平台”还有关键断点。建议不要按完成验收，应按“骨架完成，端到端闭环未通过”进入下一轮修复。

优先级最高的是：先保证前端能构建发布，再修复 Agent 产物写入和权限问题，随后开启知识入库并做存量回填。只有当知识表、向量表、图谱表、AI 产物表都有真实数据并能被页面使用时，才算真正完成这一阶段验收。
