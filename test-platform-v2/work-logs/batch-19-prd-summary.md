# Batch 19 — PRD Summary

> Product 部门 · 2026-07-19 · 回顾性文档（代码已合入 PR #36）

## 问题陈述

测试平台 v2 的**用例服务**和**接口测试**两个核心模块存在以下用户体验缺口：

### 用例服务

1. **分类管理缺失**：域（domain）和模块（module）只能通过用例自带文本字段区分，没有独立的分类管理界面。新增用例时无法限制模块选项为已维护的分类。
2. **删除不可恢复**：用例和分类采用硬删除，无法追溯或恢复。
3. **列表体验差**：搜索仅支持标题和编号，缺少模糊匹配；分页固定 20 条无选择；新增用例后不自动定位到首行。
4. **表单冗余**：新增用例包含不必要的编号和标签字段；下拉框被遮挡错位。

### 接口测试模块

1. **资产层级混乱**：服务和模块用折叠面板呈现，无法清晰区分服务边界。
2. **调试面板复杂**：完整 URL 单一输入框，无法独立设置服务器、服务、模块和路径。
3. **用例管理扁平**：所有用例平铺列表，无法按接口聚合查看。
4. **自动生成不完整**：仅覆盖前几个 Query 参数，未扩展到 Body/Path/Header。

## 成功指标

- 域/模块分类可通过独立弹窗管理（新增和删除）
- 用例搜索覆盖标题、编号、模块、前置条件
- 分页支持 20/50/100 条 + 页码跳转
- 新增用例后自动回到第 1 页并置顶
- API 测试资产可切换服务 Tab，模块默认收起
- 调试面板支持分段地址编辑
- 接口用例按接口聚合

## 非目标

- 跨项目分类共享
- 分类权限管控
- 实时协作编辑

## 用户故事

1. **测试工程师管理分类**：Given 已登录并有项目访问权限，When 点击"分类管理"按钮，Then 可以新增域和模块、删除已有分类（级联逻辑删除关联用例）。
2. **测试工程师搜索用例**：Given 用例列表页，When 输入关键字，Then 系统按标题、编号、模块、前置条件、步骤和预期结果模糊匹配。
3. **测试工程师新增用例**：Given 新增用例弹窗，When 选择域和模块并填写必填字段，Then 用例创建成功并显示在列表首行。
4. **测试工程师浏览 API 资产**：Given API 测试页，When 切换到"接口资产" Tab，Then 服务以 Tab 展示、模块默认收起、点击展开显示接口。

## 验收标准（Given/When/Then）

- **AC-1**: 域新增后立即出现在分类树；域删除后其模块和用例被逻辑删除
- **AC-2**: 关键字 "登录" 能匹配标题包含"登录"或用例编号包含"login"的记录
- **AC-3**: 新增用例弹窗默认状态为"草稿"，模块/步骤/预期结果必填
- **AC-4**: 资产 Tab 切换服务后仅显示该服务下的模块
- **AC-5**: 调试面板切换环境仅更新服务器地址，不改变服务/模块/路径
- **AC-6**: 接口用例按 api_spec_ref 聚合成可折叠组
