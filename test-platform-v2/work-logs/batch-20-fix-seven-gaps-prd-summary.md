## 4. 用户故事 + 验收标准

### US-1: 分类删除级联逻辑删除
- As a 测试工程师, I want 删除域/模块时关联用例自动标记为已删除, so that 列表数据与分类结构保持一致
- **验收**：Given 存在域 D 下有 M 模块下有 N 条用例 / When 删除域 D / Then D、M、N 条用例 `is_deleted=true`，列表不再展示

### US-2: 接口测试域用例隔离
- As a 测试工程师, I want 用例列表不展示接口测试域的用例, so that 功能用例和接口用例职责清晰
- **验收**：Given 数据库存在 `domain='接口测试'` 的用例 / When 打开用例列表 / Then 不展示这些用例

### US-3: 优先级独立列
- As a 测试经理, I want 列表中优先级是独立列, so that 我可以快速扫描 P0/P1 用例分布
- **验收**：Given 用例列表有 P0/P1/P2/P3 不同优先级 / When 查看列表 / Then 表头有「优先级」列，Badge 在独立列中

### US-4: 移除关联引用字段
- As a 测试工程师, I want 新建用例弹窗简洁, so that 不被不需要的字段干扰
- **验收**：Given 打开新建用例弹窗 / When 查看表单字段 / Then 无「关联引用」字段

### US-5: 接口测试默认资产 Tab
- As a 接口测试人员, I want 进入模块先看接口资产, so that 快速浏览已有接口
- **验收**：Given 点击「接口测试」菜单 / When 页面加载 / Then 默认激活「接口资产」Tab

### US-6: 资产→调试参数预填
- As a 接口测试人员, I want 从资产列表点调试时自动带上服务名/模块名/路径, so that 不需要手动输入
- **验收**：Given 在接口资产中点击某接口的「调试」按钮 / When 切换到快速调试 Tab / Then URL 已拼接完成、环境默认「测试5」

### US-7: 模糊搜索覆盖全字段
- As a 接口测试人员, I want 搜索接口资产时覆盖服务名/模块名/路径, so that 通过任意关键词都能找到目标接口
- **验收**：Given 输入 "camel-service" / When 搜索 / Then 返回该服务下所有接口

### US-8: 路径显示 `/` 转 `-`
- As a 接口测试人员, I want 层级路径用 `-` 分隔, so that 视觉上区分层级结构
- **验收**：Given 接口路径 `/ee/search/synonyms/cou` / When 在资产列表中展示 / Then 显示为 `ee-search / synonyms-cou`

### US-9: 接口资产备注独立字段
- As a 接口测试人员, I want 每个接口有独立备注字段, so that 记录接口用途/注意事项
- **验收**：Given 查看接口资产列表 / When 展开接口详情 / Then 可见「备注」列/字段
