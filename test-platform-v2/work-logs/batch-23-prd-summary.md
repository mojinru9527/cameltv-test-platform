# batch-23 PRD Summary — 接口测试模块优化

> 日期：2026-07-20 | Product 部门 | 7 项需求

## 问题陈述

当前接口测试模块（`/apitest`）存在 7 个用户体验和数据质量缺陷：

1. 接口资产列表默认全部展开，服务多时页面过长不易导航
2. 缺少服务维度的 Tab 筛选，用户需在混杂列表中找到目标服务
3. Service → Module → Endpoint 层级关系不够明确
4. 快速调试的响应结果位置不直观，需手动滚动
5. 接口用例列表平铺展示，同一接口的多条用例散落各处
6. 快速调试的 URL 地址未拆分，环境切换不联动服务器地址
7. AI 自动生成的接口用例覆盖度不足，边界值/类型/组合场景缺失

## 成功指标

- 资产列表默认收起后，首次加载页面高度减少 60%+
- 服务 Tab 切换后，用户可在 1 次点击内定位目标服务
- 用例按接口聚合后，同一接口的用例集中展示
- 用例生成数量：参数≥3 时 ≥25 条，参数≥5 时 ≥40 条

## 非目标

- 不改变后端 API 接口签名
- 不引入新的 UI 组件库
- 不修改数据库 schema

## 用户故事

### US-1: 资产列表层级导航
**作为** 测试工程师
**我要** 在接口资产列表中看到 Service → Module → Endpoint 的层级结构，且默认收起
**以便** 快速定位目标接口，不被大量展开的列表淹没

验收标准：
- Given 接口资产列表已加载，When 页面首次渲染，Then 所有服务名和模块名均为收起状态
- Given "全部服务" Tab 选中，When 展开某服务，Then 显示该服务下的模块列表（收起状态）
- Given 某服务 Tab 选中，When 查看列表，Then 只显示该服务下的模块（收起状态）
- Given 模块已展开，When 查看接口，Then 接口按 pathGroup 分组显示

### US-2: 用例按接口集合
**作为** 测试工程师
**我要** 在接口用例列表中看到按接口聚合的用例组
**以便** 快速了解每个接口有多少条用例，并批量执行

验收标准：
- Given 用例列表已加载，When 查看列表，Then 用例按所属接口分组，显示 `[METHOD] /path` 格式的组头
- Given 用例组，When 展开组，Then 显示该接口下的所有用例

### US-3: 调试响应位置
**作为** 测试工程师
**我要** 在发送请求后自动滚动到响应结果区域
**以便** 第一时间看到响应状态和数据

验收标准：
- Given 快速调试页面，When 请求完成返回结果，Then 页面自动滚动到响应结果面板

### US-4: 地址与环境联动
**作为** 测试工程师
**我要** 服务器地址与服务名/模块路径/接口路径拆分显示，切换环境时服务器地址联动更新
**以便** 清晰了解 URL 各部分并在环境间快速切换

验收标准：
- Given 快速调试页面，When 页面加载，Then 默认选中"测试5"环境并填充服务器地址
- Given 切换环境，When 选择不同环境，Then 服务器地址自动更新
- Given 已填写各段地址，When 发送请求，Then URL 由四段拼接而成

### US-5: 用例生成覆盖增强
**作为** 测试工程师
**我要** 自动生成的接口用例覆盖更多参数类型和边界值
**以便** 减少手动补充用例的工作量

验收标准：
- Given 接口有≥3 个请求参数，When 自动生成用例，Then 至少生成 25 条用例
- Given 接口有≥5 个请求参数，When 自动生成用例，Then 至少生成 40 条用例
- Given 请求参数有不同类型，When 自动生成用例，Then 包含 null/空值/零值/负数等边界覆盖
