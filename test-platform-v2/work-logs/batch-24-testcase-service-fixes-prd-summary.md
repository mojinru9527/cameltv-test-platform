# Batch 24 — PRD Summary
> **Product (🟦)** | Date: 2026-07-20 | Status: Approved

## 1. 问题陈述

测试平台用例服务模块存在 10 个用户反馈问题，涉及列表展示、筛选举止、模块分类管理和 API 稳定性：

1. **列表冗余**：用例列表展示"编号"字段，占用横向空间但用户不关心系统编号
2. **列顺序不直观**：模块名称应在用例标题之前，方便按模块扫读
3. **步骤信息不可读**：前置条件/操作步骤/预期结果在表格中连贯拼接，无法分辨每一步
4. **排序违反直觉**：新增用例排在末页末尾，用户需翻页查找刚创建的用例
5. **删除域报 400**：点击删除域时 API 请求到 `/domains/undefined`，域 ID 丢失
6. **模块分类无层级**：删除功能未按域→模块两级展示，用户无法逐级管理
7. **筛选缺默认值**：域/模块/优先级下拉无"全部"默认选项，新用户不知如何查看全部
8. **新增域/模块 405**：POST `/domains` 返回 405 Method Not Allowed，域/模块新增功能不可用
9. **接口测试域冗余**：接口测试域独立展示但与业务域混排，接口用例应归入接口测试模块
10. **搜索文案+范围不清**：搜索框 placeholder 不清楚，且模糊搜索范围不明

## 4. 用户故事 + 验收标准

### US1: 用例列表信息清晰
- As a 测试工程师, I want 用例列表隐藏系统编号、模块名在前、步骤分步展示, so that 快速扫读和定位用例
- 验收：Given 用例列表页 / When 加载完毕 / Then 无"编号"列；模块名称列在用例标题左侧；前置条件/操作步骤/预期结果每步独立成行，带"1、2、3、"编号

### US2: 新增用例第一时间可见
- As a 测试工程师, I want 新增用例排在列表第一条, so that 无需翻页即可确认创建成功
- 验收：Given 创建一条新用例 / When 返回列表 / Then 该用例显示在第一页第一行

### US3: 模块分类管理可用
- As a 项目管理员, I want 新增/删除域和模块功能正常、按域→模块两级展示, so that 维护项目用例分类体系
- 验收：Given 模块分类管理弹窗 / When 新增域→成功，新增模块→成功，展开域→可删除模块，直接删除域→成功 / Then 无 405/400 错误；接口测试域不在列表中

### US4: 筛选体验符合直觉
- As a 测试工程师, I want 域/模块/优先级默认显示"全部"，搜索框提示清晰, so that 首次使用时无需猜测操作方式
- 验收：Given 用例列表页 / When 首次加载 / Then 三个下拉默认显示"全部域/全部模块/全部优先级"；搜索框 placeholder 为"搜索标题/关键字"

## 5. 技术考量

- **依赖**：`R.err()` 类方法不存在是导致 405/500 的根因——Python `AttributeError` 被 FastAPI 转为 500 或路由异常
- **已知风险**：`get_category_tree` 的 fallback 路径不返回 domain ID，导致前端 `categoryId()` 返回 null，删除按钮禁用——但若用户绕过 UI 直接调 API 会触发 `undefined` URL
- **待解决问题**：接口测试域删除后，已有 API 用例的 domain 字段需手动迁移
