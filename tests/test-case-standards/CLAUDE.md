# tests/test-case-standards — 测试用例标准中心

> CamelTv 所有测试用例的统一编写规范。10 篇标准覆盖功能测试、接口测试、UI 测试、缺陷管理的全流程。

## 核心原则

1. **所有用例必须遵循本目录标准** — 无论是 AI 生成还是人工编写
2. **功能用例三要素** — 前置条件 + 操作步骤 + 预期结果（明确可验证）
3. **接口用例三要素** — 入参校验 + 业务逻辑校验 + 返回值校验
4. **优先级体系** — P0（核心必测）→ P1（常用）→ P2（一般）→ P3（次要）

## 优先级定义

| 级别 | 含义 | 覆盖要求 |
|------|------|---------|
| **P0** | 核心功能，每版本必测 | 100% 覆盖，自动化优先 |
| **P1** | 常用功能，影响大部分用户 | 自动化优先，至少冒烟覆盖 |
| **P2** | 一般功能，影响部分用户 | 时间允许时覆盖 |
| **P3** | 边缘场景，影响极少数用户 | 探索性测试为主 |

## 用例编号规范

```
TC-{模块缩写}-{序号:03d}

示例:
  TC-LOGIN-001  登录模块第 1 条用例
  TC-API-AUTH-001  接口测试-鉴权模块第 1 条
  TC-UI-HOME-001   UI 自动化-首页第 1 条
```
