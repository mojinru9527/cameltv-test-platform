**一、功能性**

|  |  |
| --- | --- |
| 测试点分类 | 测试点 |
| 功能点缺失、不一致 | 系统实现的功能少于需求中的内容 |
| 系统实现功能与需规中描述的功能不能一一对应 |
| 系统实现了需求概设中没有的功能但是功能实现有误 |
| 功能实现错误 | 图中、表中逻辑计算错误，结果和计算公式计算结果不符 |
| 开始时间大于结束时间，系统没有给出提示 |
| 开始时间等于结束时间应该可以正常返回，而不是提示报错信息 |
| 导出EXCEL文件，所导出的文件名称为乱码，导出的表头、字段、内容和页面展示的不一致 |
| 系统title名称与系统界面名称不一致 |
| 弹出窗口，弹窗名称的正确性，应和按钮保持一致或意义一致 |
| 数据缺少必要的单位信息 |
| 必填的输入信息未标注或提示 |
| 导入表格格式和模板不符合excel时应该有提示信息，且导入失败 |
| 进行二次查询，查询结果未按查询条件展示 |
| 必填项不输入数据时，单击保存按钮，可以保存成功 |
| 上传附件，单击保存按钮，系统缺少成功的提示信息 |
| 系统提示语信息描述不易用户理解 |
| 上传文件格式限制需要和需求一致，不一致时应该上传不成功且有错误提示 |
| 系统缺少注销功能 |
| 系统的提示语不明确，不简明 |
| 环境搭建错误、数据错误 | 缺少测试数据，如表中、图中等所有测试需要的数据 |
| 图中、表中展示出的数据不一致，存在矛盾 |
| 测试所需要的服务、插件等未正确安装导致功能错误或不可测 |
| 数据明显不符合逻辑，如千焦和焦耳换算错误等 |
| 需要连接外网的系统，未连接外网导致的部分功能无法测试 |
| 系统稳定性 | 系统异常退出 |
| 系统崩溃 |
| 系统停止服务 |
| 程序无法运行 |
| 插件下载 | 若需要特殊插件或软件，如BPA、PSSE。则系统应提供插件的下载地址并能成功下载 |

**二、易用性**

|  |  |
| --- | --- |
| 测试点分类 | 测试点 |
| 响应慢 | 修改数据后，单击确认按钮，确认时间过长 |
| 页面加载时间过长 |
| 系统整体卡顿 |
| 界面 | 页面展示不美观 |
| 文本框显示不全 |
| 字符串显示不全，出现“...”，且鼠标移动过去没有弹窗显示完整名称 |
| 超长字符时，超出边框展示 |
| 拓扑图中文字描述部分排版凌乱无法识别 |
| 统一边框显示、边框对齐 |
| 显示部分被遮挡 |
| （按钮、操作图标、链接）摆放位置不统一 |
| 系统风格不一致，如删除时弹窗的风格不一致等 |
| 无法修改的输入项建议置灰 |
| 如期、年份选择控件不统一 |
| 按钮上的名称能一行显示的不能折行显示 |
| 除非信息系统有特殊要求，否则不得使用横向滚动条 |
| 系统上所有菜单不得超过三级（除非有特殊要求） |
| 可终止性 | 终止执行功能缺失，执行时无法回退。如缺少“取消”按钮，用户操作重要功能时无法撤回 |
| 在线帮助 | 在线帮助功能缺失、错误或不完善 |
| 易操作性 | 弹窗应该“确认”在左，“取消”在右 |
| 时间选择框无法清空已有的数据 |
| 新增数据后，新增对话框应自动关闭 |
| 鼠标指向按键没有高亮提示，单击按键没有手型动态效果 |
| 快捷键与常用快捷键冲突 |
| 用户非法操作导致的系统错误 |
| 用户执行删除功能时无提示 |
| 打开菜单后，菜单下拉栏应自动关闭 |
| 输入校验 | 输入的手机号码、身份证、url地址、邮箱等信息格式准确性的验证 |
| 输入范围不合理，如新建或修改员工信息时人员年龄最小值为2岁以上，与日常不符 |
| 输入没有长度限制提示，输入超长字符后可以保存成功，但是会只保留规定长度的字符 |
| 数据唯一性校验，如人名，电话、文件编号、身份证号等。重复应该有报错 |
| 有些输入项，不输入为空值时也能保存成功，需要查看下是否需要改为必填 |
| 输入框输入特殊字符@#￥%……&\*（除文字、字母、数字之外所有字符）的校验，除非有特殊要求，否则禁止输入 |
| 查询框输入特殊字符@#￥%……&\*（除文字、字母、数字之外所有字符），应有提示信息，或查不出结果 |
| 输入的信息中有表情符号，应不能保存成功 |
| 超长字符输入时应有提示，且无法保存成功 |
| 查询时输入“NULL”，无法查询出符合条件的结果信息 |
| 新增数据为“NULL”，单击保存，系统报错 |
| 输入时对空格应限制 |
| 易理解性 | 输入框的提示不正确 |
| 提示信息不应该出现“系统内部错误”、“请联系管理员”等无说明指导意义的信息 |
| 默认值可用性 | 常用数据应自动填入默认值 |

**三、兼容性**

|  |  |
| --- | --- |
| 测试点分类 | 测试点 |
| 浏览器 | 浏览器不兼容低版本ie9、chrome。高版本ie11、chrome。还有firefox，safari等。若需要特定只支持X浏览器的X版本，需要需求概设中注明，且在提交电科院测试时基础平台软件清单里浏览器项注明“否”，备注里写只支持X浏览器的X版本，然后再补充一份基础平台软件清单不符合说明。 |
| 分辨率 | web需要对常见显示器、笔记本电脑分辨率可以自适应，app需要支持不同品牌、不同屏幕大小、分别率的终端设备 |
| 操作系统 | web系统需要测试win7和win10的兼容性，app若支持IOS，则需要测试Android和IOS |
| 其他软件 | 与其他软件不兼容/兼容，主要为杀毒软件 |
| 接口 | 与需求概设中涉及的其他系统的接口 |

**四、可维护性**

|  |  |
| --- | --- |
| 测试点分类 | 测试点 |
| 角色划分 | 用户角色未划分或不清晰明了、职责不明确 |
| 易分析性 | 系统日志开关功能未实现，标准时前台还是后台实现？ |
| 错误日志未提供（错误日志即程序异常可以提供错误原因提示信息并易于用户定位问题），形式可以为后台或前台 |
| 运行日志未提供（运行日志即提供运行时的状态监控记录，包括信息系统失败的原操作等统计信息），形式可以为后台或前台 |
| 系统日志开关功能未实现 （哪张表哪个字段，0关闭，1开启） |
| 未提供标准格式的告警日志，如告警时间，告警类型、告警级别、告警状态、告警模块、告警标题、告警详细信息、发生次数、最初告警时间、重复次数等。展示方式为前台展示 |
| 未提供登录日志，登录日志只需要isc上展示 |
| 维护操作 | 系统重启困难、重启方法复杂 |

**六、可移植性**

|  |  |
| --- | --- |
| 测试点分类 | 测试点 |
| 适应性 | 软件是否可移植到其它硬件、软件环境中运行 |
| 易安装性 | 是否易于安装、卸载。安装，软件是否可以一键安装，或自定义安装等；卸载，软件是否可以一键卸载等 |
| 易替换性 | 软件是否易于升级替换，包括：升级，是否可以在线升级，是否可以自动升级；覆盖安装，是否可以覆盖安装；数据复用，软件被替换后，原数据是否仍然可以使用。 |
| 依从性 | 产品或系统遵循与可移植性相关的标准、约定或法规以及类似规定的程度。这点跟其他特性一样，也就是软件的功能是否符合相关标准、法规对可移植性方面的要求 |

**七、文档**

|  |  |
| --- | --- |
| 测试点分类 | 测试点 |
| 完整性 | 主要测试文档内容是否齐全，有没有内容的遗漏 |
| 正确性 | 文档的编写有没有错误，除了内容之外，还包括文档的格式、语法、拼写 |
| 一致性 | 文档中相同部分的内容，前后是否存在不一致和矛盾的地方 |
| 易理解性 | 主要是看文档是否容易让读者读懂，比较典型的是一些技术类的文档，存在大量的缩略语，这些缩略语一定要进行清晰的说明 |
| 易浏览性 | 针对文档的结构，如文档中会显得跳转内容 |
