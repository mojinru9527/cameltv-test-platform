# 03-视频延迟测试

> **模块说明**: 本模块用于测试直播视频从主播端推流到观众端播放的端到端延迟时间。是音视频质量测试中最核心的指标。
> **前置条件**: 已完成 [01-环境安装指南](01-环境安装指南.md) 和 [02-测试素材准备](02-测试素材准备.md)
> **测试工具**: OBS Studio、Bandicam、opencv.py、analysis.py

---

## 1. 测试原理（小白必读）

```
[主播端OBS推流] → [直播服务器] → [观众端播放]
     ↑                                ↑
  发送时间戳视频                  录制屏幕画面
                                     ↓
                              OCR识别时间戳
                                     ↓
                          对比发送时间 vs 接收时间 = 延迟
```

一句话概括：**用主播端播放的"带时间戳的视频"去推流，观众端录制下来，用OCR识别画面中的时间戳，计算差值就是延迟。**

## 8. 常见问题

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| OCR识别不到时间戳 | 时间戳区域ROI不对 | 调整opencv.py中的ROI坐标（第30行附近） |
| OCR识别率很低 | 图像质量差/时间戳字体太小 | 增大时间戳字体，或使用高对比度颜色 |
| 延迟数据波动很大 | 网络抖动 | 延长录制时间（120秒以上），取更多采样点 |
| analysis.py报错 | 缺少依赖库 | `pip install pandas matplotlib numpy` |
| 录制的视频没有时间戳画面 | 视频播放/推流设置问题 | 检查OBS媒体源是否循环播放，视频是否正常 |
