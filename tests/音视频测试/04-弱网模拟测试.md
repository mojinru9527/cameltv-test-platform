# 04-弱网模拟测试

> **模块说明**: 本模块用于模拟弱网环境（高延迟、丢包、低带宽），验证直播在各种恶劣网络条件下的表现。通过Network Emulator工具精确控制网络参数。
> **前置条件**: 已完成 [01-环境安装指南](01-环境安装指南.md) 和 [02-测试素材准备](02-测试素材准备.md)
> **测试工具**: NEWT (Network Emulator for Windows Toolkit)、OBS Studio、Bandicam、opencv.py、analysis.py

---

## 1. 测试原理

```
[OBS推流] → [NEWT弱网模拟] → [直播服务器] → [观众端播放]
                 ↑
         在这里人为注入：
         - 延迟 (Latency)
         - 丢包 (Packet Loss)
         - 抖动 (Jitter)
         - 带宽限制 (Bandwidth Limit)
```

## 3. 弱网配置模板

### 3.1 配置一：轻度弱网（模拟WiFi信号弱）

| 参数 | 设置值 | 说明 |
|------|--------|------|
| 配置名称 | `轻度弱网_WN01` | |
| 延迟(Latency) | `100ms` | 正常网络通常<30ms |
| 丢包率(Packet Loss) | `1%` | 每100个包丢1个 |
| 抖动(Jitter) | `20ms` | 延迟波动范围 |
| 带宽限制(Bandwidth) | `2Mbps` | 限速2M |
| 应用方向 | ☑ 上行(Uplink) ☑ 下行(Downlink) | |

**预期表现**：
- 延迟增加 200-500ms
- 画面偶有卡顿，但基本可观看
- 音频偶有断续

### 3.2 配置二：中度弱网（模拟隔墙+距离远）

| 参数 | 设置值 | 说明 |
|------|--------|------|
| 配置名称 | `中度弱网_WN02` | |
| 延迟(Latency) | `200ms` | |
| 丢包率(Packet Loss) | `3%` | 每100个包丢3个 |
| 抖动(Jitter) | `50ms` | |
| 带宽限制(Bandwidth) | `1Mbps` | |
| 应用方向 | ☑ 上行(Uplink) ☑ 下行(Downlink) | |

**预期表现**：
- 延迟增加 500-1000ms
- 画面卡顿频繁，偶有花屏
- 音频断续明显

### 3.3 配置三：重度弱网（模拟地铁/电梯场景）

| 参数 | 设置值 | 说明 |
|------|--------|------|
| 配置名称 | `重度弱网_WN03` | |
| 延迟(Latency) | `500ms` | |
| 丢包率(Packet Loss) | `5%` | |
| 抖动(Jitter) | `100ms` | |
| 带宽限制(Bandwidth) | `500Kbps` | 仅500K |
| 应用方向 | ☑ 上行(Uplink) ☑ 下行(Downlink) | |

**预期表现**：
- 延迟增加 1000-2000ms
- 画面严重卡顿，频繁花屏/黑屏
- 音频严重断续或丢失

### 3.4 配置四：极端弱网（模拟信号即将中断）

| 参数 | 设置值 | 说明 |
|------|--------|------|
| 配置名称 | `极端弱网_WN04` | |
| 延迟(Latency) | `1000ms` | |
| 丢包率(Packet Loss) | `10%` | 每10个包丢1个 |
| 抖动(Jitter) | `200ms` | |
| 带宽限制(Bandwidth) | `200Kbps` | |
| 应用方向 | ☑ 上行(Uplink) ☑ 下行(Downlink) | |

**预期表现**：
- 可能直接断流
- 如果不断流，延迟>3000ms，画面不可看

| 步骤 | 操作 | 详细说明 | 完成 |

| 4 | OBS开始推流 | 使用时间戳测试视频 | ☐ |
| 5 | 等待推流稳定 | 约10秒 | ☐ |
| 6 | 观众端进入直播间 | 测试手机打开APP→进入直播间 | ☐ |
| 7 | 开始录制 | Bandicam录制`[____]`秒（建议120秒，弱网下需更长采样） | ☐ |
| 8 | 停止录制 | 保存为 `WN01_recording_[编号].mp4` | ☐ |
| 9 | 停止推流 | OBS点击停止 | ☐ |
| 10 | 关闭NEWT规则 | 恢复网络正常 | ☐ |
| 11 | 重复2次 | 每个配置需测3次取平均值 | ☐ |

## 8. 常见问题

| 问题 | 解决方法 |
|------|----------|
| NEWT无法启动 | 以管理员身份运行；检查是否有其他网络代理软件冲突（VPN、抓包工具等） |
| NEWT规则不生效 | 确认规则已启用；检查是否选对了网络接口（WiFi/以太网） |
| 弱网下录制文件过大 | 正常现象（弱网下画面变化大，编码效率降低），确保磁盘空间>5GB |
| 启用弱网后完全断网 | 检查规则是否配置在正确的接口上；降低丢包率重试 |
