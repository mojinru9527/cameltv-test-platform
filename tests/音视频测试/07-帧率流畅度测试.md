# 07-帧率流畅度测试

> **模块说明**: 本模块用于测试直播视频的帧率和流畅度，检测是否存在丢帧、卡顿、画面不流畅等问题。通过播放跑马灯/移动标记测试视频来量化流畅度。
> **前置条件**: 已完成 [01-环境安装指南](01-环境安装指南.md) 和 [02-测试素材准备](02-测试素材准备.md)
> **测试工具**: OBS Studio、Bandicam、fluency_test_60fps.mp4 (或 fluency_test_pattern.mp4)

---

## 1. 测试原理

```
[主播端播放跑马灯视频]               [观众端录制]
 白色竖条匀速从左到右移动                ↓
 (60fps下，每帧移动固定像素)      逐帧分析录制画面
       ↓                          检测白条移动是否:
 如果某帧白条位置跳跃过大            - 匀速 (流畅)
 或静止不动 = 发生了丢帧/卡顿        - 跳跃 (丢帧)
                                     - 静止 (卡顿)
```

### 3.2 录后目视检查

用视频播放器以 **0.5倍速** 或 **0.25倍速** 回放录制的视频，观察跑马灯白条：

| 观察时间点 | 白条移动状态 | 是否卡顿 |
|------------|-------------|----------|
| 0-30秒 | ☐ 匀速 ☐ 轻微抖动 ☐ 明显跳跃 | ☐ 是 ☐ 否 |
| 30-60秒 | ☐ 匀速 ☐ 轻微抖动 ☐ 明显跳跃 | ☐ 是 ☐ 否 |
| 60-90秒 | ☐ 匀速 ☐ 轻微抖动 ☐ 明显跳跃 | ☐ 是 ☐ 否 |
| 90-120秒 | ☐ 匀速 ☐ 轻微抖动 ☐ 明显跳跃 | ☐ 是 ☐ 否 |

> **判断标准**: 
> - 匀速 = 白条平滑移动，无跳帧感 → 流畅
> - 轻微抖动 = 白条移动不够平滑，偶尔跳一下 → 有轻微丢帧
> - 明显跳跃 = 白条一跳一跳的，不连续 → 丢帧严重

## 4. 定量分析（选做，需要额外脚本）

### 4.1 逐帧分析法

创建 `C:\直播测试脚本\frame_analysis.py`：

```python
import cv2
import numpy as np
import sys

def analyze_fluency(video_path):
    """分析视频流畅度：检测跑马灯白条位置变化"""
    cap = cv2.VideoCapture(video_path)
    
    prev_bar_pos = None
    frame_num = 0
    jumps = []  # 记录跳跃帧
    stills = []  # 记录静止帧
    
    fps = cap.get(cv2.CAP_PROP_FPS)
    print(f"视频FPS: {fps}")
    
    while True:
        ret, frame = cap.read()
        if not ret:
            break
        
        # [可调整参数] 跑马灯白条检测区域
        height, width = frame.shape[:2]
        roi = frame[300:400, :]  # 白条所在高度
        gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)
        
        # 找到最亮的列（白条位置）
        col_sums = np.sum(gray, axis=0)
        bar_pos = np.argmax(col_sums)
        
        if prev_bar_pos is not None:
            diff = bar_pos - prev_bar_pos
            expected_move = 4  # [可调整] 60fps下每帧预期移动像素
            
            if abs(diff) < 1:  # 静止
                stills.append(frame_num)
            elif abs(diff) > expected_move * 2:  # 跳跃
                jumps.append((frame_num, diff))
        
        prev_bar_pos = bar_pos
        frame_num += 1
        
        if frame_num % 1000 == 0:
            print(f"分析进度: {frame_num}帧")
    
    cap.release()
    
    total_frames = frame_num
    print(f"\n=== 流畅度分析结果 ===")
    print(f"总帧数: {total_frames}")
    print(f"静止帧数: {len(stills)} ({len(stills)/total_frames*100:.2f}%)")
    print(f"跳跃帧数: {len(jumps)} ({len(jumps)/total_frames*100:.2f}%)")
    print(f"流畅帧数: {total_frames - len(stills) - len(jumps)} ({(total_frames - len(stills) - len(jumps))/total_frames*100:.2f}%)")
    
    if stills:
        print(f"\n卡顿发生位置(帧号): {stills[:20]}..." if len(stills) > 20 else f"\n卡顿发生位置(帧号): {stills}")
    if jumps:
        print(f"\n跳跃发生位置(帧号, 像素差): {jumps[:20]}..." if len(jumps) > 20 else f"\n跳跃发生位置(帧号, 像素差): {jumps}")

if __name__ == "__main__":
    if len(sys.argv) != 2:
        print("用法: python frame_analysis.py <录制视频路径>")
        sys.exit(1)
    analyze_fluency(sys.argv[1])
```

## 7. 常见问题

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 录制视频本身卡顿 | Bandicam帧率设置过低 | 录制帧率设置为60fps，检查CPU占用 |
| 白条位置检测不准确 | ROI区域不对 | 调整frame_analysis.py中的ROI坐标 |
| 观众端画质选项不可选 | APP版本/直播间设置限制 | 检查主播端推流分辨率是否支持多档 |
| 跑马灯白条抖动但没丢帧 | 视频编码导致的画面抖动 | 区分"编码伪影"和"真正丢帧"，前者通常不影响观感 |
