## 3. 问题排查清单

### 3.1 OCR识别问题

| 现象 | 排查步骤 | 解决方法 |
|------|----------|----------|
| 识别率为0 | 1. 检查ROI区域是否包含时间戳 | 修改opencv.py中ROI坐标 |
| | 2. 检查时间戳字体/颜色是否清晰 | 使用高对比度时间戳(白字黑底) |
| | 3. 检查Tesseract是否安装正确 | 运行 `tesseract -v` 验证 |
| 识别率很低(<50%) | 1. 增加图像预处理 | 在opencv.py中ROI前添加二值化处理 |
| | 2. 调整OCR配置参数 | 修改`--psm`参数值(6/7/8/13) |
| | 3. 增大时间戳字体 | 重新生成更大字体的测试视频 |
| 识别出的时间戳格式不对 | 1. 调整字符白名单 | 修改`tessedit_char_whitelist`参数 |

### 3.2 延迟数据分析问题

| 现象 | 排查步骤 | 解决方法 |
|------|----------|----------|
| analysis.py报编码错误 | 1. 检查log文件编码 | 用记事本打开看是否乱码 |
| | 2. 尝试不同编码 | 脚本已支持utf-8和gbk自动切换 |
| 延迟数据全为0或异常 | 1. 检查log文件内容 | 确认timestamp列有数据 |
| | 2. 检查assumed_fps值 | 修改为测试视频的实际帧率 |
| | 3. 检查parse_timestamp | 确认时间戳格式能被正确解析 |
| 图表中文显示为方块 | 1. 安装中文字体 | `pip install matplotlib`后配置中文字体 |

### 3.3 测试视频播放问题

| 现象 | 排查步骤 | 解决方法 |
|------|----------|----------|
| OBS无法加载测试视频 | 1. 检查视频编码 | 转换视频为H.264编码: `ffmpeg -i input.mp4 -c:v libx264 output.mp4` |
| | 2. 检查文件路径 | 路径中不要有中文或空格 |
| 视频播放卡顿 | 1. 检查PC性能 | 关闭其他占用CPU的程序 |
| | 2. 降低视频分辨率 | 使用720p而不是1080p |
| 时间戳与视频不同步 | 1. 检查视频生成参数 | 确保FPS设置正确 |

### 3.4 弱网模拟问题

| 现象 | 排查步骤 | 解决方法 |
|------|----------|----------|
| NEWT规则不生效 | 1. 以管理员运行 | 右键→以管理员身份运行 |
| | 2. 检查网络接口 | 选择正确的网卡(WiFi/以太网) |
| | 3. 关闭其他网络软件 | 关闭VPN、抓包工具、代理软件 |
| 启用弱网后完全断网 | 1. 降低丢包率 | 从1%开始逐步增加 |
| | 2. 检查带宽限制 | 不要设置低于100Kbps |

### 3.5 录制问题

| 现象 | 排查步骤 | 解决方法 |
|------|----------|----------|
| Bandicam录制没有声音 | 1. 检查音频设置 | 设置→音频→勾选"录制系统声音" |
| | 2. 检查系统音量 | 确保系统音量未静音 |
| 录制文件打不开 | 1. 检查文件是否完整 | 确认录制正常结束，未强制关闭 |
| | 2. 尝试转换格式 | `ffmpeg -i broken.mp4 -c copy fixed.mp4` |
| 录制画面黑屏 | 1. 检查Bandicam录制区域 | 确保录制范围包含播放器窗口 |
| | 2. 检查图形加速 | 关闭硬件加速重试 |

   - 缺陷总数

   - 遗留问题
   - 建议

   - 是否建议发布
