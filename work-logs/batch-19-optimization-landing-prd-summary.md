## 4. 用户故事 + 验收标准

### US-1: API 测试 — 接口资产服务切换
- As a 测试工程师, I want 接口资产页面按服务 Tab 切换并显示模块→接口层级, so that 我能快速定位到目标接口
- 验收：Given 接口资产页面已加载 / When 点击某个服务 Tab / Then 只显示该服务的模块列表，模块默认收起，展开后显示接口列表

### US-2: API 测试 — 快速调试地址拆分
- As a 测试工程师, I want 快速调试页面将地址拆分为服务器/服务/模块/接口路径独立字段, so that 我能单独修改各部分而不重新拼接完整 URL
- 验收：Given 快速调试页面 / When 选择环境 / Then 仅更新服务器地址字段，其他字段保持不变

### US-3: API 测试 — 接口用例按接口分组
- As a 测试工程师, I want 接口用例按接口聚合且默认收起, so that 我能快速浏览接口维度的用例覆盖情况
- 验收：Given 接口用例 Tab / When 加载用例列表 / Then 同接口的用例归入一个 Collapsible 组，默认关闭，显示用例数量 Badge

### US-4: API 测试 — 自动生成覆盖全参数
- As a 测试工程师, I want AI 自动生成覆盖每个请求参数的空值/类型/边界/异常场景, so that 生成的用例能充分覆盖接口的异常路径
- 验收：Given 一个包含多个 Query/Body/Path/Header 参数的接口 / When 触发生成 / Then 每个参数至少有一条空值或类型异常覆盖，总数不超过 200

### US-5: 用例管理 — 分页选项
- As a 测试工程师, I want 用例列表支持 20/50/100 条分页切换, so that 我能根据需要调整每页显示数量
- 验收：Given 用例列表页面 / When 切换分页大小为 50 / Then 列表刷新为每页 50 条，分页器正确更新

### US-6: 用例管理 — 步骤格式化回显
- As a 测试工程师, I want 编辑用例时步骤以"1、xxx"格式回显且保留原始 JSON 结构, so that 我能直观阅读步骤内容同时不破坏数据格式
- 验收：Given 编辑一个包含 JSON 步骤的用例 / When 打开编辑抽屉 / Then 步骤显示为格式化文本（1、操作描述 — 预期结果），提交时保留原始 JSON 结构

### US-7: 需求管理 — 来源智能压缩
- As a 测试工程师, I want 需求来源列智能显示（蓝湖链接提取版本、其他来源显示域名）, so that 表格列宽可控且我能悬停查看完整 URL
- 验收：Given 需求列表含蓝湖 URL / When 渲染来源列 / Then 蓝湖链接显示为"蓝湖 v{版本号}"，其他来源显示域名，悬停可见完整 URL

## 5. 技术考量

- **依赖**：无需新增依赖，复用现有 Radix Tabs/Collapsible、Tailwind
- **风险**：修改 ApiCaseTab/DebugTab/AssetTab 可能影响现有测试执行流程
- **测试数据**：验收过程中创建的测试用例、需求、API 资产将在验收通过后删除，不影响用户数据
- **回滚**：所有改动在独立 feature 分支，合入前可随时回滚
