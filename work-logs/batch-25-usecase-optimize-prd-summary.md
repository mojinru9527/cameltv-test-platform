# Batch 25 PRD Summary — 用例服务 + 接口测试模块优化

> Product Department | 2026-07-21

## 问题陈述

用户在使用测试平台过程中，发现用例服务模块和接口测试模块存在多处体验问题：

1. **用例列表 UI 不规范**：列顺序不合理（模块名称未放首位）、隐藏"编号"字段需求、步骤内容未按序号格式化显示、用例不按时间倒序排列
2. **筛选搜索不完整**：三个筛选器（域/模块/优先级）缺少"默认全部"逻辑，搜索提示文案不准确，不支持跨字段模糊搜索
3. **模块分类管理残缺**：缺少新增域/模块的入口（导致 405 报错）、删除域/模块时 URL 拼接 `undefined` 导致 400 报错、删除未级联逻辑删除关联用例、"接口测试"域不应在用例服务中显示
4. **新建用例弹窗缺陷**：下拉框被遮挡、模块非必填、移除不需要的"关联引用"字段
5. **接口测试模块未完成**：默认不展示接口资产 tab、缺少搜索功能、三级层级不清晰、调试 URL 未正确拼接、环境切换未生效、Swagger 导入未落地、接口备注未独立显示

## 成功指标

- 用例列表：模块名称 → 用例标题 → 优先级（字段独立），每行仅一行展示，超出 "......" 截断
- 筛选：域/模块/优先级默认"全部"，搜索支持域/模块/前置条件/步骤/预期结果模糊匹配
- 模块分类：支持新增域 → 新增模块流水，删除级联逻辑删除，接口测试域隐藏
- 新建用例：弹窗下拉框不被遮挡，模块必填，移除"关联引用"字段
- 用例排序：按 `created_at` 倒序，最新用例排在第一位
- 接口测试：默认展示接口资产 tab，三级层级（服务→模块→路径），调试 URL 正确拼接，支持环境切换

## 非目标

- 不改变后端 API 的响应 envelope 结构
- 不引入新的第三方组件库
- 不修改测试计划/报告/缺陷等其他模块
- Excel/Xmind 导入导出后端不删除（仅移除前端 UI 入口），保留 API 给其他调用方

## 用户故事

### US-1: 用例列表重构
**As** 测试工程师  
**I want** 用例列表按模块名称→标题→优先级排列，每行只显示一行，新用例排在最前  
**So that** 我能快速定位最新用例并清晰看到每条用例的核心信息

**验收标准:**
- Given 用户进入用例服务页面，When 页面加载完成，Then 列表列顺序为：复选框 → 模块名称 → 用例标题 → 用例等级（独立 Badge） → 前置条件（编号格式） → 操作步骤（编号格式） → 预期结果（编号格式） → 评审 → 操作
- Given 列表中存在多条用例，When 用户查看列表，Then 每条用例仅占一行，超出宽度用 "......" 截断
- Given 用户新建一条用例，When 保存成功返回列表，Then 新用例显示在列表第一条
- Given 列表中存在用例，When 页面加载，Then 不显示"编号"列（case_id）

### US-2: 筛选搜索增强
**As** 测试工程师  
**I want** 三个筛选器默认"全部"状态，搜索支持跨字段模糊匹配  
**So that** 我不需要手动清空筛选器就能看到全部数据，搜索一个关键字就能在所有相关字段中匹配

**验收标准:**
- Given 用户进入用例服务页面，When 页面加载完成，Then 域/模块/优先级筛选器均显示默认"全部"选项且已选中
- Given 用户只选择域不选模块，When 点击搜索，Then 返回该域下所有模块的用例
- Given 用户只选模块不选域，When 点击搜索，Then 返回该模块的用例
- Given 用户输入关键字"登录"，When 点击搜索，Then 在标题/域/模块/前置条件/步骤/预期结果中模糊匹配"登录"
- Given 搜索框，When 页面加载，Then placeholder 显示"搜索标题/关键字"

### US-3: 模块分类管理
**As** 测试工程师  
**I want** 能新增/删除域和模块，删除时级联逻辑删除关联用例，接口测试域不显示  
**So that** 我能完整管理用例的分类体系，且删除分类不会留下孤立的可见用例

**验收标准:**
- Given 用户打开模块分类管理弹窗，When 输入域名称点击新增，Then 新域出现在域列表中（POST /domains 返回 200）
- Given 已存在某域，When 用户在域下新增模块，Then 新模块出现在该域的模块列表中
- Given 用户删除某域，When 确认删除，Then 该域及其所有模块和关联用例被标记为 is_deleted=true
- Given 用户删除某模块，When 确认删除，Then 该模块及其关联用例被标记为 is_deleted=true
- Given 模块分类树，When 页面渲染，Then 接口测试域不出现在列表中
- Given 域/模块树，When 展开某域，Then 显示其下所有模块名称+用例数，可逐模块删除

### US-4: 新建用例弹窗修复
**As** 测试工程师  
**I want** 新建用例弹窗下拉框不被遮挡，模块为必填，无冗余字段  
**So that** 我能顺利完成用例创建，不会因为下拉框被遮挡或选漏必填项而创建失败

**验收标准:**
- Given 新建用例弹窗打开，When 点击域/模块/优先级下拉框，Then 下拉选项完全可见，不被弹窗边缘遮挡
- Given 新建用例表单，When 用户未选择模块点击保存，Then 提示"请选择模块"阻止提交
- Given 新建用例表单，When 页面渲染，Then 不显示"用例编号"和"关联引用"字段（移除 api_spec_ref）

### US-5: 接口测试模块重构
**As** 测试工程师  
**I want** 接口测试默认展示接口资产 tab，资产按三级层级显示（服务→模块→路径），支持搜索和调试  
**So that** 我能快速浏览和搜索接口资产，从资产跳转调试时自动拼接完整 URL

**验收标准:**
- Given 用户点击"接口测试"菜单，When 页面加载，Then 默认展示"接口资产" tab
- Given 接口资产列表，When 数据加载，Then 以服务为第一级、模块为第二级、路径为第三级的层级展示
- Given 接口路径 `/ee/search/synonyms/cou` 属于服务 `camel-service`，When 显示时，Then 模块显示为 `ee-search`，路径显示为 `synonyms-cou`（用 `-` 替换 `/`）
- Given 接口资产搜索框，When 用户输入关键字，Then 在服务名/模块名/路径中模糊匹配
- Given 用户从接口资产点击调试，When 跳转到快速调试，Then 自动拼接完整 URL：`{环境base_url}/{服务名}/{模块路径}/{接口路径}`
- Given 快速调试页面，When 用户从接口资产跳入，Then 默认环境为"测试5"
- Given 快速调试页面，When 用户直接进入（非从资产跳转），Then 所有字段为空，由用户手动填写
- Given 接口资产列表，When 显示每条接口，Then "备注"字段独立显示为一个列/行
- Given Swagger 导入弹窗，When 用户输入服务名+Swagger URL 并确认导入，Then 自动生成服务名/模块名/接口路径存入平台
- Given 调试模式下切换环境，When 用户选择不同环境，Then 仅变更测试地址（base_url），服务名/模块名/路径不变

### US-6: 清理冗余功能
**As** 测试工程师  
**I want** 用例服务移除 Excel/Xmind 导出下载按钮  
**So that** 页面更简洁，减少不必要的操作入口

**验收标准:**
- Given 用例服务页面，When 页面渲染，Then 不显示 Excel 导出和 Xmind 导出按钮
- Given 脑图页面，When 页面渲染，Then 移除 Xmind 导出按钮

### US-7: 逻辑删除一致性
**As** 测试工程师  
**I want** 删除域/模块时，列表中关联的用例同时被标记删除；接口用例从用例列表中移除  
**So that** 用例列表中的用例始终与分类体系保持一致，不会出现幽灵数据

**验收标准:**
- Given 某域下有 5 条用例，When 删除该域，Then 这 5 条用例的 is_deleted=true，不再在列表中显示
- Given 用例列表，When 选择"接口用例" tab，Then 不再显示接口用例（case_type=api 的用例需从用例服务隐藏）
- Given 现有接口用例，When 执行逻辑删除处理，Then 接口用例迁移到接口测试模块管理
