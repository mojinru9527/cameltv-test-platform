# Batch 26 — PRD Summary

> **Product (🟦)** | Date: 2026-07-21 | Status: Draft

## 1. 问题陈述

### 1.1 用例服务 5 个 UI 缺陷（直接影响日常使用）

| # | 问题 | 用户影响 | 根因 |
|---|------|---------|------|
| 1 | 点击重置后「全部模块」下拉框无法下拉 | 重置后无法再按模块筛选，必须刷新页面 | `selDomain=''` → `selModules=[]` → Select 只有 1 个 option → Radix Select 拒绝打开 |
| 2 | 全部域/全部模块/全部优先级下拉框错位 | 下拉菜单飞到奇怪的位置，体验差 | 过滤器 SelectContent 未指定 `position="popper"`，默认 item-aligned 在固定高度容器中偏移 |
| 3 | 用例列表行高与左侧模块分类不一致 | 视觉不对齐，页面看起来粗糙 | 表格行无 min-height，自然紧凑 vs 树节点有固有高度 |
| 4 | 鼠标悬停表格时底部出现横向滚动条 | 每次悬停就闪现滚动条，非常干扰 | 表格外层 `overflow-x-auto` + `min-w-[900px]` 导致 hover 触发 overflow |
| 5 | 新建用例弹窗的「基本信息」和「评审」tab 大小不一致 | 切换 tab 时弹窗跳动，体验割裂 | 新建 form `max-h-[60vh]` vs 编辑 tabs `max-h-[50vh]` |

### 1.2 需求文档-蓝湖证据采集（工作流断裂）

| # | 问题 | 用户影响 | 根因 |
|---|------|---------|------|
| 1 | 任务列表不可见，关闭侧边栏就找不到 | 用户不知道任务跑没跑、进度如何 | 证据任务只有 Sheet 侧边栏（`LanhuEvidenceJobDrawer`），无持久化任务列表 |
| 2 | 任务不自动执行，需手动点刷新 | 流程断裂，用户困惑"为什么点了开始采集没反应" | Worker 5s 轮询已就绪，但前端无自动刷新机制 |
| 3 | 任务状态/进度/历史不可追溯 | 无法查看发起了几次、哪次失败、为什么失败 | 无任务列表持久化展示 |
| 4 | 采集完成后看不到完整文档内容 | OCR 文字无法替代图片为主的蓝湖原型 | 只有文本提取，无截图预览能力 |
| 5 | 功能拆分不能体现完整需求 | AI 只拿到 OCR 文本碎片，丢失原型视觉信息 | 蓝湖以图片为主、文字为辅；OCR 文本不能完整描述交互 |

### 1.3 版本迭代（核心痛点）

| # | 问题 | 用户影响 | 根因 |
|---|------|---------|------|
| 1 | 同一蓝湖链接多版本，每次都全量处理 | 浪费 AI token、时间，重复劳动 | 无版本检测机制 |
| 2 | 新版本不知道改了什么 | 产品/测试无法聚焦变动范围 | 无从对比两个版本的页面差异 |
| 3 | 功能拆分每次都从头做 | 之前确认过的功能点又要重审 | 无继承机制 |

### 1.4 知识中心版本同步（新需求）

| # | 问题 | 用户影响 | 根因 |
|---|------|---------|------|
| 1 | 蓝湖版本更新后知识库不感知 | 知识库停留在旧版本，检索到过期内容 | 知识入库仅在首次上传触发，无版本变更通知 |
| 2 | 版本间差异未沉淀为知识 | 无法追溯"v14.1.0 → v14.2.0 改了什么" | 变更检测器只做 content_hash 对比，无结构化 diff |

## 5. 技术考量

### 5.1 依赖关系

```
Phase 1 (任务可见) ──→ Phase 2 (预览+对比) ──→ Phase 3 (AI增强)
                              ↓
                    知识中心版本同步
```

- Phase 1 独立可交付，不依赖 Phase 2/3
- Phase 2 依赖 Phase 1 的任务面板（预览入口从任务面板进入）
- Phase 3 依赖 Phase 2 的 diff_json（AI 需版本差异上下文）
- 知识中心版本同步依赖 Phase 2 的 diff_json

### 5.2 已知风险

| 风险 | 缓解 |
|------|------|
| 感知哈希在深色/浅色主题下误判 | 截图前强制统一主题，对比阈值调优 |
| difflib 对中英文混排相似度不准 | 先分词再对比，中文字符级相似度 |
| 页面匹配失败（page_name 变更） | 降级策略：按 page_path + 结构相似度匹配 |
| 版本号解析失败（非标准 URL） | 降级为 initial（不对比），不阻断主流程 |
| 知识库同步可能产生大量新切片 | 限制单次差异入库的最大切片数 |

### 5.3 待解决问题

- 蓝湖 page_id 在版本间是否稳定？（需实测验证）
- 感知哈希的汉明距离阈值需要 A/B 测试确定
