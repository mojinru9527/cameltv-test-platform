## 2. 逐模块差距分析

### 2.1 🔴 音视频专项 (`/special`) — P0 阻断

| 维度 | 现状 | 目标 |
|------|------|------|
| 指标数据 | `random.uniform` 伪造 | ffprobe/ffplay 真实拉流探测 |
| 对 camel1.tv 的影响 | **致命** — 体育直播平台的核心测试需求无法验证 | 起播时延、卡顿率、首帧时间、画质切换等真实指标 |

- **Backlog 状态**：未列入（三引擎做真批次五未覆盖音视频真实化）
- **CLAUDE.md**：🧪 演示态
- **实操确认**：仍是随机数

### 2.2 🔴 API 测试 (`/apitest`) — P0/P1 缺口

**已做真**：后端 `httpx` 真实发 HTTP 请求，8 种断言类型，环境变量，数据驱动。

**仍缺失**：

| 缺口 | 优先级 | 说明 |
|------|--------|------|
| 生产环境保护 | P0 | 执行 API 测试时需有环境标记/拦截，防止误伤线上 |
| 请求快照 | P1 | 失败时无法复现（缺完整 request/response 记录） |
| 任务取消真中断 | P1 | 取消只改状态，不中断执行循环 |
| Swagger 导入 | P1 | 无法从 Swagger JSON/YAML 自动生成接口用例 |
| 请求历史团队共享 | P2 | 当前仅 localStorage，不可跨用户 |

### 2.3 🔴 UI 自动化 (`/uitest`) — P0/P1 缺口

**已做真**：通过 `npx playwright test` 真实执行 Playwright 脚本。

**仍缺失**：

| 缺口 | 优先级 | 说明 |
|------|--------|------|
| 产物回看 | P0 | 截图/视频/trace 字段占位，不可实际查看 |
| 异步执行 | P1 | 同步阻塞，长时间跑会超时 |
| 脚本资产管理 | P1 | 无法团队协作管理 .spec 文件 |
| 环境注入 | P1 | 无法注入 baseURL/凭据 |
| 用例数量 | P1 | 仅 2 条，远不足以做生产回归 |
