## 5. 结论

**初版评估: 符合预期，有明确改进方向**

测试平台 v2 已覆盖「需求→用例→计划→执行→报告」管理闭环，RBAC、多项目隔离、AI 用例生成等核心能力已生产可用。主要差距集中在：

# Batch 37 — 模块联动架构设计

> **Design (📝)** | Date: 2026-07-23 | 本 Batch 只设计，不编码

## 1. 背景

当前测试平台各模块独立运作，测试同学需要在不同模块间手动切换操作：
- 在需求管理导入需求文档
- 手动复制需求内容到用例服务创建用例
- 再手动去 API 测试模块编写接口用例
- 再手动去 UI 自动化模块创建回归脚本

目标：一个需求文档 → 自动触发 → AI 功能拆分 → AI 用例生成 → 关联接口测试 → 关联 UI 自动化回归。

## 2. 联动架构

```
需求文档导入
      │
      ▼
┌──────────────┐    AI 功能拆分     ┌──────────────┐
│  需求管理     │ ─────────────────→ │  知识中心     │
│ /requirement  │                    │ /knowledge    │
└──────────────┘                    └──────┬───────┘
                                          │
                              ┌───────────┼───────────┐
                              │           │           │
                              ▼           ▼           ▼
                     ┌──────────┐ ┌──────────┐ ┌──────────┐
                     │ AI用例生成│ │ 接口匹配  │ │ 功能关联  │
                     │ DeepSeek │ │ Swagger  │ │ 版本比对  │
                     └────┬─────┘ └────┬─────┘ └────┬─────┘
                          │            │            │
                          ▼            ▼            ▼
                    ┌──────────┐ ┌──────────┐ ┌──────────┐
                    │  用例服务  │ │ API 测试 │ │UI 自动化 │
                    │ /testcase │ │ /apitest │ │ /uitest  │
                    └──────────┘ └──────────┘ └──────────┘
```

## 3. 数据流

### 3.1 阶段一：需求 → 知识沉淀

```
POST /api/v1/requirements/upload
  → 解析文档（Word/Excel/Xmind）
  → 提取功能点/模块/接口引用
  → 生成知识切片 → 写入知识中心（knowledge_domain=project）
  → 返回提取摘要
```

### 3.2 阶段二：知识 → AI 用例生成

```
POST /api/v1/cases/ai-generate
  body: { requirement_id, knowledge_source_ids[], generation_mode }
  → 从知识中心检索关联切片
  → DeepSeek LLM 生成用例（Given/When/Then 格式）
  → 按优先级 P0-P3 分级
  → 返回用例草稿列表
  → 测试同学审阅 → 确认入库
```

### 3.3 阶段三：用例 → 接口测试联用

```
前置条件: Swagger/OpenAPI 文档已导入

POST /api/v1/apitest/link-from-case
  body: { case_ids[], openapi_source_id }
  → 匹配用例中的 API 端点与 Swagger 路径
  → 自动生成请求模板（method/url/headers/body）
  → 填充测试数据（从用例步骤推断）
  → 返回接口用例草稿列表
```

### 3.4 阶段四：用例 → UI 自动化关联

```
获取上一个版本已上线功能列表
  → GET /api/v1/release-bundles/latest/features

POST /api/v1/uitest/link-from-features
  body: { feature_ids[], case_ids[] }
  → 为每个功能生成 Playwright 测试脚本骨架
  → 填充选择器/断言（基于用例步骤）
  → 返回 UI 自动化脚本草稿
```

## 4. 前提条件清单

| 联动路径 | 前提条件 | 状态 |
|---------|---------|:----:|
| 需求→知识 | 需求文档导入功能 | ✅ 已有 |
| 知识→AI用例 | 知识检索 API + LLM Key | ✅ 已有 |
| 用例→接口测试 | **Swagger/OpenAPI 文档已导入** | ⚠️ 需测试同学上传 |
| 用例→UI自动化 | **上一个版本功能列表** | ⚠️ 需版本管理模块完善 |
