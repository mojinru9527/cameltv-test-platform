## 1. 知识中心是什么

知识中心是 CamelTv 测试平台的**统一知识管理中枢**。它不是一个简单的文档库，而是一个 **AI 驱动的知识生产系统**（A-System）：

```
信息输入 → AI 自动加工 → 结构化知识 → 知识关联图谱 → 可检索可复用
```

**核心能力：**

| 能力 | 说明 |
|------|------|
| 📥 **自动沉淀** | 需求文档上传、接口导入、用例创建、测试执行失败 → 自动入库 |
| 🤖 **AI 加工** | 自动向量化（RAG 可检索）、自动实体提取、自动关系建图 |
| 🔍 **混合检索** | 向量语义搜索 + 关键词全文检索 + RRF 融合排序 |
| 🗺️ **知识图谱** | 实体-关系可视化，支持自演化（合并重复、发现隐含关系） |
| ✅ **保鲜治理** | 新鲜度评分、定期验证、废弃标记、迭代快照 |
| 🧠 **Skills 模板** | 预置 6 个 AI 能力模板：生成用例、分析缺陷、提取契约、检测矛盾、总结迭代、预测回归 |

**两类知识域：**

- 🏀 **项目知识**（`knowledge_domain=project`）：需求、接口定义、测试用例、缺陷、执行记录——为你的体育项目服务
- 🔧 **平台研发知识**（`knowledge_domain=platform`）：踩坑记录、设计决策（ADR）、最佳实践、问题模式——为测试平台自身研发服务

## 2. 快速入门：5 分钟上手

### 第一步：进入知识中心

登录测试平台 → 左侧导航栏 → 点击「知识中心」。你会看到 12 个功能 Tab。

### 第二步：看看你项目中已经有什么

默认打开「**项目知识**」Tab，显示所有已自动沉淀到 PARA 分类的项目知识。

### 第三步：搜点东西试试

切到「**检索**」Tab，输入关键词（如"支付"、"直播"、"登录"），体验混合检索。

### 第四步：探索知识图谱

切到「**图谱**」Tab，看看你的项目知识已经自动构建了怎样的实体关系网络。

### 第五步：捕获一条灵感

点击页面右下角的 **⊕** 浮动按钮，快速记录一个想法。AI 会自动加工分类。

## 3. 功能详解

### 3.1 项目知识 Tab

**路由：** `/knowledge?tab=project`

**功能：** 以 PARA 分类视角展示项目相关知识的卡片视图。

- **按 batch（批次）自动分组**：每个批次的 PRD、测试计划、执行结果会聚合在一起
- **新鲜度色标**：
  - 🟢 绿色（>80%）：知识保鲜度高，可信
  - 🟡 黄色（40-80%）：需要关注，建议近期验证
  - 🔴 红色（<40%）：可能过时，建议验证或废弃
- **卡片点击** → 打开知识源详情弹窗（含原始内容 + AI 切片 + 元数据）

**数据来源：** 自动从需求导入、接口导入、用例创建、测试执行等环节自动沉淀。无需手动上传。

**适合谁：** 测试人员、QA、产品经理——查看项目的测试知识资产。

---

### 3.2 平台研发 Tab

**路由：** `/knowledge?tab=platform`

**功能：** 展示测试平台自身研发过程中积累的知识。

按 PARA 分类分段展示：

- **Active（活动区）**：当前正在处理的改进、优化事项
- **Resource（资源区）**：
  - 🔴 **问题模式**：反复出现的问题及解决方案（如 "DeepSeek 安全分类器不可用"）
  - 📋 **设计规范**：ADR 架构决策记录、技术方案
  - ✅ **最佳实践**：经过验证的推荐做法
- **Archive（归档区）**：历史记录、已解决的旧问题

**数据来源：**
- Work-logs 批量导入（91 个历史工件）
- Agent Team 执行完成后自动入库
- 手动捕获的灵感（经 AI 加工后分到此类）

**适合谁：** 平台开发者、架构师——避免重复踩坑，查阅设计决策。

---

### 3.3 概览仪表盘 Tab

**路由：** `/knowledge?tab=overview`

**功能：** 知识中心全局健康一览。

| 指标 | 说明 |
|------|------|
| 知识源总数 | 按 source_type 分类统计（需求/接口/用例/缺陷/手动等） |
| RAG 状态 | 嵌入模型健康检查、向量库是否就绪 |
| 实体/关系 | 知识图谱中的实体数和关系数 |
| AI 产物 | 待审核/已采纳/已驳回的 AI 生成产物 |
| 保鲜评分分布 | 整体知识新鲜度直方图 |
| 最近迭代 | 最近的迭代和快照概览 |

**适合谁：** 项目管理者、Team Leader——全局视角掌握知识资产状况。

---

### 3.4 智能检索 Tab

**路由：** `/knowledge?tab=search`

**功能：** 对知识库进行混合检索（Hybrid RAG）。

- **输入关键词/问题** → AI 同时执行：
  - 向量语义搜索（理解语义，找到意思相近的内容）
  - 关键词全文搜索（精确匹配）
  - RRF（Reciprocal Rank Fusion）融合排序
- **返回结果**：最相关的知识片段 + 来源链接 + 相关度评分
- **搜索健康检查**：右上角可查看 RAG 嵌入模型是否正常

**搜索技巧：**
- 用自然语言提问效果更好："登录接口有哪些安全要求？"
- 而不是零散关键词："登录 安全"

**适合谁：** 所有人——日常工作中快速查找已有知识。

### 3.5 知识源管理 Tab

**路由：** `/knowledge?tab=sources`

**功能：** 管理所有知识源条目（底层数据视图）。

| 操作 | 说明 |
|------|------|
| 🔍 **类型筛选** | 按需求/接口/用例/缺陷/手动等过滤 |
| 📄 **分页浏览** | 每页 20 条，支持翻页 |
| 👁️ **查看详情** | 点击「查看」→ 弹窗展示完整内容 |
| ✅ **验证保鲜** | 点击 ✓ 按钮 → 更新 last_verified_at 和 freshness_score |

**详情弹窗包含：**
- 标题、类型、PARA 分类、知识域、版本、状态
- 保鲜评分（百分比 + 色标）
- 最后验证时间、创建时间、更新时间
- 来源链接
- 元数据 JSON
- **原始内容**（完整展示，可滚动）
- **AI 切片列表**（每片含类型、标题、token 数、完整内容）

**适合谁：** 知识管理员、高级用户——精细管理知识资产。

### 3.6 AI 审核台 Tab

**路由：** `/knowledge?tab=artifacts`

**功能：** 审查 AI 自动生成的产物，决定采纳或驳回。

| 操作 | 说明 |
|------|------|
| 👁️ **查看** | 浏览 AI 生成的产物内容 |
| ✅ **采纳** | 确认 AI 产物正确，标记为 approved |
| ❌ **驳回** | AI 产物不可用，标记为 rejected，附驳回原因 |
| 📥 **导入用例库** | 将 AI 生成的测试用例正式导入用例管理系统 |

**工作流：**
```
AI 自动生成产物 → 进入审核台 → 人工审核 → 采纳/驳回 → 导入正式库
```

**适合谁：** QA 工程师——把控 AI 生成质量，人机协作。

### 3.7 知识图谱 Tab

**路由：** `/knowledge?tab=graph`

**功能：** 可视化知识实体关系网络。

**画布操作：**
- 🖱️ 拖拽平移、滚轮缩放
- 🟢 **放大/缩小/适应** 按钮（画布右上角）
- 🔍 **点击节点** → 右侧显示详情（类型、描述、置信度）
- 🎨 **图例过滤** → 点击图例可隐藏/显示特定类型的实体

**实体类型颜色：**
- 🔵 蓝色 = API 接口
- 🟢 绿色 = 字段
- 🟣 紫色 = 需求
- 🟠 橙色 = 测试用例
- 🔴 红色 = 缺陷

**工具栏按钮：**

| 按钮 | 功能 |
|------|------|
| **提取** | 手动触发实体提取与关系建图（通常 AI 自动做） |
| **演化** | 概念地图自演化：合并重复实体、更新置信度、发现隐含关系 |

**演化结果面板：** 显示合并重复实体数、置信度更新数、新发现隐含关系数。

**自动调度：** 每天凌晨 04:13 自动执行一次演化。

**适合谁：** 所有人——直观理解知识之间的关联关系。

## 4. 体育项目怎么用

### 4.1 自动沉淀（你不需要手动做什么）

知识中心会**自动**从你的日常工作流中沉淀知识：

| 你的操作 | 自动沉淀什么 |
|----------|-------------|
| 上传需求文档 | 需求文本 → 向量化 → 可检索 |
| 从蓝湖导入原型 | 原型内容 → 结构化需求知识 |
| 导入 Swagger/OpenAPI | API 定义 → API 实体 + 字段实体 |
| 创建测试用例 | 用例内容 → 关联到需求和 API |
| 执行测试计划 | 执行结果（通过/失败/跳过）→ 关联到用例 |
| 提交缺陷 | 缺陷信息 → 关联到用例和需求 |

### 4.2 日常使用场景

**场景 1：新需求来了，想看看有没有类似的历史用例**
→ 打开「检索」，输入需求关键词 → 找到相关用例和接口定义 → 复用

**场景 2：版本发布前，想确认测试覆盖**
→ 打开「图谱」→ 查看需求→用例→API 的关系链路 → 确认无遗漏

**场景 3：线上出了 Bug，想追溯根因**
→ 打开「检索」，搜索 Bug 关键词 → 找到历史类似缺陷 → 关联分析

**场景 4：写测试报告，需要数据支撑**
→ 打开「概览」→ 查看统计数据 → 引用到报告中

**场景 5：新人接手老项目，需要快速了解**
→ 打开「Wiki 知识库」→ 浏览项目知识全貌 → 查看知识图谱

**场景 4：想了解某类问题的历史模式**
→ 打开「Skills」→ 执行「分析缺陷模式」→ AI 自动分析

## 6. PARA 知识组织体系

知识中心使用 **PARA** 方法对知识进行分类组织：

| 分类 | 含义 | 适用场景 | 示例 |
|------|------|---------|------|
| 📥 **inbox** | 待处理 | 刚捕获的灵感、未分类的知识 | 快速捕获的想法 |
| 📁 **project** | 项目相关 | 有明确截止日期的任务 | 某版本的测试计划 |
| 📂 **area** | 责任区 | 持续关注但无截止日期 | API 测试规范、问题模式 |
| 📚 **resource** | 参考资料 | 可复用的知识资产 | ADR、设计规范、最佳实践 |
| 🗄️ **archive** | 归档 | 已废弃或不活跃 | 旧版本的测试用例 |

**AI 会自动将 inbox 中的知识加工并分类到合适的 PARA 类别。**

## 7. AI 自动加工流水线

每一条知识进入系统后，会自动经过以下处理：

```
① 文本清洗
   ↓
② 知识源入库（去重）
   ↓
③ 智能切片（按语义分割）
   ↓
④ 向量嵌入（fastembed BGE-small-zh）
   ↓
⑤ 实体提取（LLM 提取 API/字段/需求/用例/缺陷）
   ↓
⑥ 关系建图（实体间的 contains/depends_on/covers 等）
   ↓
⑦ AI Agent 自动标注（按需触发）
```

**你唯一需要做的就是正常使用平台**——其他都是自动的。

## 8. 常见问题 FAQ

### Q: 知识库的数据会丢失吗？
A: 知识库基于 SQLite（可升级 PostgreSQL），所有数据持久化存储。迭代快照机制提供额外的版本保护。

### Q: AI 自动加工错了怎么办？
A: 打开「AI 审核台」→ 驳回错误产物。打开「知识源」→ 手动修改分类或废弃。

### Q: 怎么让知识保鲜？
A: 定期点击知识源的「✓ 验证」按钮。系统也会通过保鲜评分提醒你哪些知识需要关注。

### Q: 图谱里的实体和关系是怎么来的？
A: LLM（DeepSeek）自动从知识源中提取实体和关系。你可以通过「AI 审核台」控制质量。

### Q: 灵感捕获的内容去哪了？
A: 进入 inbox（待处理），然后 AI 自动加工分类到合适的 PARA 类别。

### Q: 如何搜索效果最好？
A: 用自然语言提问，而不是关键词。例如「支付接口的测试用例有哪些？」比「支付 用例」效果好。

### Q: 知识库会不会影响平台性能？
A: AI 加工在独立后台线程执行（SessionLocal 隔离），不影响正常使用。
